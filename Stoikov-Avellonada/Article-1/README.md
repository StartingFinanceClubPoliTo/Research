# Market Making Simulation: Naive vs Avellaneda-Stoikov

**Project:** [Stoikov-Avellonada](../)  
**Article:** 1  
**Notebook:** [`codice_articoloSF.ipynb`](./codice_articoloSF.ipynb)  
**Main output:** `Avellaneda_Stoikov_Output.mp4`

This folder contains the reproducibility material for the first article in the Stoikov-Avellonada project. The notebook builds a cinematic simulation of the Avellaneda-Stoikov optimal market-making model, combining a 3D surface of the optimal bid spread with a four-panel trading simulation.

## What It Shows

- **Optimal quoting surface** - a 3D visualization of the bid spread as a function of time and inventory.
- **Price and quotes simulation** - a stylized mid-price path with bid and ask quotes generated by the model.
- **Inventory dynamics** - the market maker's inventory path during the simulation.
- **Cash and PnL tracking** - cash balance and profit-and-loss evolution across the rendered scenario.
- **Video rendering pipeline** - frame generation, parallel rendering, and MP4 compilation with a dark financial-dashboard style.

## Files

- **`codice_articoloSF.ipynb`** - end-to-end notebook containing both the complete pipeline and a step-by-step explanation of the implementation.

## Requirements

Python >= 3.10 is recommended.

Install the required packages with:

```bash
pip install numpy matplotlib imageio imageio-ffmpeg
```

The notebook uses only local simulation logic and does not require external market data.

## How to Reproduce

1. Open `codice_articoloSF.ipynb`.
2. Run the complete pipeline cell, or follow the explanatory section step by step.
3. The script renders frames in `temp_avellaneda_frames/`.
4. The final video is exported as `Avellaneda_Stoikov_Output.mp4`.
5. Execution logs are written to `avellaneda_pipeline.log`.

## Implementation Notes

- The simulation is configured for 1920x1080 output at 30 FPS.
- The default duration is 13 seconds, with a short final hold.
- Rendering uses Python multiprocessing through `Pool`; adjust `CONFIG["N_WORKERS"]` if your machine has fewer available CPU cores.
- Matplotlib runs with the `Agg` backend, so the notebook can render frames without opening GUI windows.
